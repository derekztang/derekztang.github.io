This class doesn't do much at the moment. We can define an initialize method that gives certain characteristics to the football that are created. Let's say in our game, football games start at game 1

class football
def initialize
@game = 1
end
end

The @ marks a variable as an instance variable. This means that it is a variable that belongs to a single instance, as oppose to across an entire class. In this case, football game doesn't interfere with another game, since all games are different instances. It also means that any method inside the class football can access that variable.

So what exactly makes a football game? it consists of rules.
