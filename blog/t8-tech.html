Recursive Functions

Go Recurse Yourself

Mar 27, 2015

Recursion is a personal fascination. It's a programming paradigm hailed by many as an elegant approach to solve problems. To approach a problem via recursion is to break that problem into a sequence of increasingly smaller instances of the same problem. To be more concrete, a recursive function is one that calls itself, thereby repeating sequentially and reducing the problem each time. Like looping, it entails the repeated execution of a portion of code. However, it differs in that each recursion reduces the complexity of the problem, whittling it down to its solution. For example, let's use this strategy to write a Ruby method that does basic exponentiation.

A recursive method. It calls ITSELF.

This method first checks if the exponent is 0. If so, it returns 1. This is known as the base case. It's where the repetition ends. It then multiplies the base by the output of, well, itself with the exponent decremented by one. The method will get repeated until the exponent eventually homes in on 1, stopping the repetition. In other words, the solution to 24 is trimmed down to 23 which is trimmed to 22 and so on.

This program can also be written by using a simple loop or a #times method. In fact, doing so would produce a program that executes roughly 10 times faster. Writing a recursive function can sometimes come at a performance cost. In some contexts, this is compensated for by a dramatic improvement in the readability of the code. Let's look at an example that is drastically simplified by using recursion: finding the determinant of a matrix.

The determinant is a useful number associated with a square matrix in linear algebra. For this method we'll use the method of minors. The input for the method will be a 2D array, each inner array representing one row of the matrix from top to bottom.

A method for calculating the determinant of a matrix.

In a manner analogous to the exponent method, it first checks if the matrix is 2x2. If it is, the simple cross multiplication algorithm is used. Otherwise, the make_minor method is used to map each of the minors to a new array with the appropriate coefficients. These terms are then added using #inject to arrive at the determinant! As we can see, the recurring problem is the determinant of each minor. The size of the minors decreases by once each time, eventually converging to 2, where the repetition terminates. This method is nice because it works for any size of square matrix!

Hopefully I've convinced you of the elegance and simplicity endowed on a program with a judicious use of recursion. Using a loop would make the matrix_determ method far more complicated. Although it's an expensive approach, it may be worth it to refine your code. For more discussion of recursive functions, see chapter 3 of Eloquent JavaScript.
