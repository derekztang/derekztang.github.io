Enumerable Methods

The Power of Container Objects

Feb 22, 2015

The Enumerable module is one of the hallmarks of Ruby. It's an assortment of methods for container objects, like arrays and hashes which I talked about in my previous blog post. These methods streamline the process of performing operations on each of the elements within a container object. The crux of Enumerable is that its inclusion into a class requires an #each method for that class, because enumerable methods call #each. Think of #each as a set of instructions for how to iterate over an object. Once that is set, you can set in motion the pre-existing machinery for enumerable objects! For example the Array#each method looks something like this:

The Array#each method in action.

Arrays have a very intuitive progression. Each element in the array gets yielded to a code block in order from 0, 1, 2, until the end. In the example above, #each is called to puts each integer in the array incremented by one. One of the most useful Enumerable methods is #map. This stores each output in a new array.

Array#mapping to a new array.

The advantage here is that the array can be saved to a variable and manipulated, instead of just being putsed. Again, #map calls #each. The same is true for other Enumerable methods like #cycle, #select, or #group_by. The beauty of these methods is that they exist for all other container object classes, like Range and Set, although they don't all work exactly the same way. They can even be applied to classes of your own creation, as long as you define an #each method! Check out some other Enumerable methods in the Ruby Docs.
