Ruby vs. JavaScript Loops

What Are You Getting Loopy For?

Mar 19, 2015

Ruby is a very high-level language, meaning you can express a lot with only a few lines of code. This is merited by its myriad of built in functionality. The power in its methods make Ruby programs petite and elegant, much like their brilliant gemstone namesake. Rarely do other languages beat the beauty and brevity of Ruby. However, this level of refinement comes with a performance cost. Ruby programs are slower to execute and they tend to scale poorly. JavaScript is a faster language that comes close to Ruby's ritzy methods. JavaScript is also far more widely used, so it's valuable to know.

Although JavaScript doesn't match Ruby's highly numerous, reliable Enumerable methods, there are things that it does better than Ruby (besides speed). Essentially every one of Ruby's Enumerable methods can be emulated in JavaScript using a for loop. Shown below is an example of a for loop being used.

Using a for loop to calculate the sum of all integers from 1 to 100.

In the parentheses after for is where the magic happens. The first expression establishes a counter variable, i, by convention. The second expression decides when the loop is over: when it evaluates to false. The third expression determines how we're counting, i.e. it doesn't necessarily count up by one after each iteration. These for loops are ubiquitous in JavaScript programs! Ruby has for loops as well, but they are less modular. They are typically defined by a range and can only increment by one. Let's compare them by using each to write a program to insert commas into an integer.

Can we write a function that takes a number and returns a string with commas properly inserted into the number?

JavaScript has no built-in method to directly insert characters into a string at a certain index. However, it can be done with an array of characters. Therefore we can call String() followed by split() to convert the number to an array. We'll then use a for loop to repeat the splice() method.

A JavaScriptfor loop in action.

After converting back to a string, it's complete! But how would we do the same thing in Ruby? As I mentioned before, Ruby for loops must increment by one, so it will be less straightforward.

The same net effect using Ruby.

Although Ruby allows us to operate directly on a string, the for loop is a bit... clumsier. We had to do a calculation to stop the loop in the right place. We also had to do a calculation on i inside the body of the loop. The pattern and intent in the JavaScript example is much clearer. Much more coercion is needed to use a Ruby for loop for this challenge.

So even though Ruby has the most inherent functionality, it doesn't have the most elegant answer to everything. The fundamentals are best handled by simpler languages.
