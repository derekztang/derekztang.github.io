Aliases

Workflow Part III

Apr 3, 2015

If you've read my previous two workflow posts (Completions and Snippets), you've noticed a recurring theme. There are things we repeat needlessly when programming. One of the cardinal sins in programming is repetition. A central tenant of this craft is writing D.R.Y. (don't repeat yourself) code. For this post, let's focus on the terminal. If you use the terminal a lot, you've more than likely typed things like cd .. a million times. You probably have some directories you navigate to a lot. Fortunately, the terminal supports aliases. Aliases are shortened versions of terminal inputs. For example, suppose I want to navigate three folders down from my home directory.

I go to this directory on a daily basis. It's laborious to type all that out, even with auto-complete. It doesn't make much sense to set that location as the default working directory either, because I often go to other directories. I'll set up an alias!

Typing sip is equivalent to pushd ~/Desktop/Projects/HappyHourHunter. With only three keystrokes I can navigate from anywhere to HappyHourHunter! To create an alias, you'll need to go to your home directory and open a (hidden) file .bash_profile. Pick your favorite text editor and edit it. I use Vim.

Scroll down to where is says # Useful aliases. This is where you can assign your alias.

You can see my sip alias, among others that expedite navigation. I have gimp in case I want to open an image with Gimp instead of the default image editor. I even have an alias, al, for creating new aliases! They've greatly streamlined the use of the terminal. Over time I've learned to love the terminal.

These workflow techniques are all a part of programming philosophy. It's all about abstraction and eliminating repetition. For my next post I'll depart from the D.R.Y. mantra and show you how to make your Sublime Text look pretty!
